Image-to-ASCII

Jan 4 2023

Usage#

  • install PIL: pip install pillow
  • put the images to convert in /data. they’ll be output to /out
  • at size choice, type 0 if no preference
  • to override size limits, type -1 on the output quality dialogue.

Motivations#

A few years ago, I found out about donut.c which was equal parts artistry and wizardry to me. I really liked the look of it, growing up surrounded by ASCII art on obscure forums.

Recently, reminded of this, I wrote a script to make similar kinds of art.

Notes#

I do not output the values as .txt files or output text to the terminal, since that doesn’t support colors (which I, personally, wanted). One potential option is outputting it as HTML which can then be embedded, but that’s a bit tedious. Maybe some day.

I used pillow to output to an image instead. In a sense this is pseudo-ascii, you have no access to the actual text that’s generated.

How it works: chunks of differing n*n pixels (depending on size choice) get their average brightness mapped to a certain ASCII character.

I use this character set, but it’s easy enough to change: $@B%8&WM#*oahkbdpqwmZO0QLCJUYXzcvunxrjft/\|()1{}[]?-_+~<>i!lI;:,\"^`'.

donut.c only uses .,-~:;=!*#$@!

This can potentially generate massive images so default parameters sacrifice details on larger scales to control output size.
